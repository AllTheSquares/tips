Random Tips

Hosting your own website on GitHub

  1. Begin by creating a new repository. Name it as followed: "username".github.io (Replace "username" with your GitHub username, you can see this in the web address of this page.
  2. I recommend that you download GitHub Desktop at this point. It makes editing your site easier than having to publish changes via a terminal. Once downloaded, clone your repository via this app.
  3. Cloning a repository creates a folder on your computer. On your favorite code editor of choice (I recommend either Visual Studio Code or Sublime Text, which may need a license to use), create a new file named "index.html amd save it to the folder of your repository on your computer (should be in the GitHub folder under Documents).This is your main page.
  4. On GitHub Desktop, it should automatically recognize any changes made to your folder. All you have to do from now on is to click on "Commit to main", and after that, to publish, click on "Publish Branch." GitHub Pages should activate by default.

Extra tips on GitHub

  1. Making changes on your site is easy enough. Just edit the html file created on the folder on your code editor of choice, commit to the main folder, and push to the origin via GitHub Desktop.
  2. If you're going to make multiple pages, there are two ways to do it: making a new repository (and giving it any name), or creating another HTML file.
  3. If you choose to make a new repository for another page, repeat the process of making a site, just make sure to go to your repository's settings on GitHub, click on Pages, and select the folder of origin (should be root). Creating another HTML file is easy enough: create another file that contains the code to your alternate page, and push it to the origin. You'll need to find a way to link to it, but the easiest way to access it is through your web address (username.github.io/example). Replace example with either the name of your new repository or your html file (witt the extension).
  4. Before publishing, I suggest you run the HTML file on your computer. It will work similar as if you were to pulbish, with the perk that you can instantly you can see any changes you make every time you refresh.
